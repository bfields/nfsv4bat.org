Interoperability testing

CIFS client, NFSv4 client, multiprotocol server. Some preliminary results:
  • For an ACL with a single EVERYONE@ ACE with various mode-bit combinations, set from one side, query from the other. (Works OK.)
  • Set basic OWNER@, GROUP@, EVERYONE@ ACL, query from both clients. (Windows translates OWNER@ and GROUP@ to current file owner and group; otherwise OK.)
  • Windows Explorer, as expected, sometimes reorders allow and denies.
  • Test handling of suid, sgid, sticky bits.
  • Identified some implementation bugs.
