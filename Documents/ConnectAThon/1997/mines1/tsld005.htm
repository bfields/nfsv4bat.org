Design Requirements

  • Require Little or No Changes to Existing Applications.

  • Be Consistent with Java Development Style

  • Extend File Namespace to Include URL’s

  • Completely Backwards Compatible

  • 100% Java

  • Extensible in Both File System Interface and Protocol Domain Space

  • Protocol Neutral (This is not an NFS proposal )

  • Support for Traditional File Based Applications as Well as URL Based Applications
